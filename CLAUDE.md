# christiangeorgelucas/xpath-tools — Axiom node package

This is an Axiom node package (language: **typescript**). Axiom turns plain typed
functions into deployed, auto-scaling endpoints — you build the whole package
with the `axiom` CLI. This file orients an AI coding agent; follow the loop and
respect the sharp edges. (Generated by `axiom init`; ADR-085.)

## What's here

- `axiom.yaml` — the manifest (package name, version, language, nodes, imports, secrets).
- `messages/messages.proto` — the protobuf type vocabulary (node inputs/outputs).
- `nodes/` — one typed function per node. `gen/` — generated bindings.

## The node signature is frozen (ADR-001)

Every node takes `AxiomContext` (the platform handle) plus one typed input and
returns one typed output. For typescript:

```
export async function myNode(ax: AxiomContext, input: InputMsg): Promise<OutputMsg>
```

## The authoring loop

1. Define messages FIRST — `axiom create message InputMsg` / `OutputMsg`, then edit `messages/messages.proto` to add fields.
2. Scaffold each node — `axiom create node MyNode --input InputMsg --output OutputMsg`. ALWAYS pass `--input`/`--output` (omitting them prompts interactively and hangs headless).
3. Implement the function body in `nodes/`.
4. `axiom generate` — compile messages into `gen/` bindings.
5. Validate — `axiom validate --json`. Fix each `ok:false` check (the `detail` tells you how); loop until `passed:true`.
6. Run locally — `axiom dev` (serves on :8083, JSON↔protobuf bridge), then `curl -s localhost:8083/nodes/MyNode -d '{...}'`.
7. Commit AND push to a git remote, then `axiom push --json` (tenant-private) and finally `axiom publish christiangeorgelucas/xpath-tools@<version>` (public, immutable).

## Sharp edges (where agents fail)

- **Messages before nodes** — `create node` fails if the input/output message doesn't exist yet.
- **Never rely on interactive prompts** — always pass `--input`/`--output`; always use `--json` for machine-readable output.
- **PascalCase node names** on the command line; files are snake_case (the CLI handles the file name).
- **Commit AND push before `axiom push`** — push builds from the git **remote at HEAD**, not your working tree.
- **Secrets via `ax.secrets.get("KEY")`**, never as node inputs; declare them in the node's `required_secrets`.
- **Nodes can call the internet directly** — import an SDK and call out; `ax` is for platform capabilities (secrets, telemetry), not a network jail.
- **`axiom push` is idempotent** (overwrites the tenant-private deploy); **`axiom publish` is immutable** — bump the version to re-release.

## AxiomContext (`ax`)

`ax.log` (structured logging) · `ax.secrets.get("NAME")` · `ax.reflection.flow`
(read-only graph view) · `ax.mutation.flow` (advanced). Do not create your own
telemetry exporter — it flows through `ax`.
